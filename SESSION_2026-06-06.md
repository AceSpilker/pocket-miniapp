## 一、双主题切换系统

### 1.1 功能需求
- 全局主题切换按钮：固定在页面右上角，滑动开关样式（左侧太阳☀️右侧月亮🌙）
- 自动切换：早上 8:00 - 晚上 18:30 为亮色主题，其他时间为暗色主题
- 手动切换：用户可手动切换，手动选择优先于自动
- 所有页面和组件支持双主题
- 亮色模式遵循 Apple Human Interface Guidelines

### 1.2 技术实现

**主题配置文件** (`utils/theme-config.js`)
- 定义亮色/暗色两套完整配色方案
- 亮色主题采用 Apple 标准配色（背景 #F2F2F7，文字 #1C1C1E）
- 图标配色：亮色模式使用粉彩色系（Pastel）

**主题管理器** (`utils/theme-manager.js`)
- 主题存储、切换、持久化
- 自动切换定时器（每分钟检查）
- 主题变化监听器模式
- 导航栏颜色同步

**主题切换组件** (`components/theme-toggle`)
- 滑动开关样式
- 支持点击切换
- 动画效果

## 二、功能管理模块

### 2.1 数据库设计

**features 表：**
```sql
id              INT PRIMARY KEY
name            VARCHAR(50)    -- 功能名称
icon            VARCHAR(100)   -- 图标（emoji）
icon_bg_color   VARCHAR(20)    -- 图标背景颜色
description     VARCHAR(200)   -- 功能描述
path            VARCHAR(200)   -- 跳转路径
is_enabled      BOOLEAN        -- 是否启用
is_hidden       BOOLEAN        -- 是否隐藏
required_permission VARCHAR(100) -- 所需权限
sort_order      INT            -- 排序
```

**user_feature_orders 表：**
```sql
id              INT PRIMARY KEY
user_id         INT            -- 用户ID
feature_id      INT            -- 功能ID
sort_order      INT            -- 用户自定义排序
is_home_visible BOOLEAN        -- 是否在首页显示
```

### 2.2 API 接口

**用户端：**
- `GET /features` - 获取功能列表
- `GET /features/home` - 获取首页功能（最多4个）
- `PUT /features/order` - 保存用户排序

**管理端：**
- `GET /admin/features` - 功能列表
- `POST /admin/features` - 创建功能
- `PUT /admin/features/{id}` - 更新功能
- `DELETE /admin/features/{id}` - 删除功能

### 2.3 前端页面

**功能中心** (`pages/features`)
- 功能列表展示
- 编辑模式：拖拽排序、设置首页显示
- 图标自定义背景色

**管理端功能管理** (`pages/admin/features`)
- 功能列表
- 创建/编辑功能
- 颜色选择器（预设颜色 + 自定义输入）

### 2.4 Bug 修复

- 后端 API 返回 `icon_bg_color` 字段
- 首页功能网格使用自定义背景色
- 功能中心列表使用自定义背景色
- 管理员功能列表显示自定义颜色

## 三、意见反馈模块

### 3.1 数据库设计

**feedbacks 表：**
```sql
id          INT PRIMARY KEY
user_id     INT            -- 用户ID
title       VARCHAR(100)   -- 标题
content     TEXT           -- 内容
status      ENUM           -- pending/replied/resolved
version     INT            -- 当前版本号
admin_reply TEXT           -- 最新回复
```

**feedback_versions 表：**
```sql
id          INT PRIMARY KEY
feedback_id INT            -- 反馈ID
version     INT            -- 版本号
title       VARCHAR(100)   -- 标题
content     TEXT           -- 内容
```

**feedback_replies 表：**
```sql
id          INT PRIMARY KEY
feedback_id INT            -- 反馈ID
version     INT            -- 对应版本
reply       TEXT           -- 回复内容
replied_by  INT            -- 回复人ID
```

**管理端：**
- `GET /admin/feedbacks` - 反馈列表
- `GET /admin/feedbacks/{id}` - 反馈详情
- `PUT /admin/feedbacks/{id}/reply` - 回复反馈

### 3.3 前端页面

**用户端：**
- `pages/feedback` - 反馈列表

**管理端：**

## 五、Bug 修复记录

### 5.1 功能图标背景色问题
- **问题**：数据库有 `icon_bg_color` 字段，但 API 未返回
- **解决**：在 `api/feature.py` 中添加 `icon_bg_color` 字段返回

### 5.2 内容页面未登录跳转问题
- **问题**：未登录访问内容页面跳转到登录页
- **原因**：`request.js` 默认需要认证
- **解决**：`get` 方法传入 `false` 参数，表示公开接口

### 5.3 内容页面亮色主题问题
- **问题**：内容页面亮色主题颜色不正确
- **解决**：
  - 添加 `theme-{{theme}}` 类到根元素
  - 添加 `.page.theme-light` 选择器样式
  - 引入 `van-loading` 组件
